Financial Risk Assessment Preprocessing Pipeline

A preprocessing comparison study on 150K borrower records focused on imbalance handling and reliable model readiness.

Overview

This project centered on the part of machine learning that often decides whether a model succeeds in practice: disciplined preprocessing. I designed and compared multiple preprocessing workflows for a large borrower dataset with missing values, outliers, multicollinearity, and heavy class imbalance.

What I built

  • Designed and benchmarked three preprocessing pipelines on 150K borrower records, explicitly addressing data quality and a 14:1 class imbalance.
  • Compared imputation, scaling, and resampling strategies to understand which sequence best preserved signal for downstream modeling.
  • Selected an Iterative Imputation + RobustScaler + SMOTE pipeline that achieved 0.8203 test ROC-AUC and 0.8199 cross-validation ROC-AUC.

Why it matters

  • Highlights strong data-processing judgment rather than relying only on black-box model tuning.
  • Shows careful treatment of imbalance, robustness, and generalization in a real credit-risk style problem.
  • Demonstrates how preprocessing design can be a major lever for dependable ML performance.
Project details

  • Dataset size: 150K borrower records

Real-Time Graph Analytics Pipeline | Neo4j + Kubernetes

A graph data system combining Neo4j, Docker, Kafka, and Kubernetes for scalable real-time analytics.

Overview

This project focused on building a scalable analytics workflow around graph data. I used the NYC Yellow Cab dataset to model connected information in Neo4j, then added orchestration and streaming components to support larger-scale graph analysis.

What I built

  • Developed a Dockerized Neo4j graph database on the NYC Yellow Cab dataset and structured the data for graph-style querying and traversal.
  • Implemented PageRank and breadth-first search to support large-scale graph analytics and relationship exploration.
  • Integrated Apache Kafka for streaming and Kubernetes for orchestration to create a more scalable, high-availability pipeline.

Why it matters

  • Demonstrates systems thinking beyond notebook-only analysis by combining storage, algorithms, streaming, and orchestration.
  • Shows comfort with graph analytics and platform tooling that matter in production-oriented environments.
  • Bridges data engineering and algorithmic problem-solving in one project.
Project details
